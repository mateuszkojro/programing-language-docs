Env::~Env ( )
Warning
Memory leak - not implemented garbage collector

Here is the place where language memory management needs to be considered there are basically two options:

  • Leave the memory management to the user
  • Implement some kind of garbage collection Given the fact that the language needs a runtime to execute scripts I decided on implementing the garbage collector using method of reference counting. Unfortunately time for finishing this assigment run out so for now unfortunately this place leaks memory.

Possible implementation of reference counted GC should be based on std::shared_ptr for simplicity of implementation and performance benefits that stand from using STL implementation
